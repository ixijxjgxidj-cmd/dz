# 融合版总纲：面向应急管理的地震 P/S 震相智能拾取与秒级研判系统

> 本文是整个项目的**最高蓝图**，融合两套方案：
> - **策略骨架**（来自 GPT 方案）：EEW 应急预警叙事、双层定位、4 周排期、答辩话术——负责"这仗怎么打、决赛怎么讲"。
> - **工程内核**（本项目已落地并测试的代码）：评分器、ObsPy mseed 读取、时间对齐、训练脚手架——负责"分算得准、跑得稳、成果不丢"。
>
> 一句话分工：**策略层告诉你怎么打，工程层保证枪不卡壳、准星没偏。冲一等奖两样都要。**

---

## 0. 作品定位（决赛专家分的地基）

**作品名（建议）**：面向广西—东盟区域应急管理的低算力地震 P/S 震相智能拾取与秒级研判系统。

**科学诚实的一句话表述**（务必这样写，不要写"提前预测地震"）：

> 本系统面向**地震发生后**的秒级快速监测研判场景：利用最先到达的 P 波，在破坏性更强的 S 波和面波到达前，通过 AI 模型自动识别 P/S 震相到时，并结合多台站关联，快速估计发震时刻、震中方向、震中位置与震级，为应急管理部门争取数秒到数十秒的预警窗口与辅助决策依据。

**为什么这样定位**：地震"提前准确预测"（某天某地几点几分几级）全世界都做不到，写进方案会被专家判为不科学。"震后秒级预警"才是工业上成立、答辩站得住的表达。

## 1. 双层架构：榜单冲分层 + 决赛展示层

榜单自动评分只考核 P/S 震相拾取（占初赛/复赛全部、决赛 80%）。EEW 定位/震级是决赛展示层，用于专家评分那 20%，**不参与榜单**。两层必须分清，避免把"单条无位置信息波形算不出震中"的硬伤吹成能力。

```
┌─────────────────────── 榜单冲分层（官方自动评分）───────────────────────┐
│  mseed 波形 → 质控/预处理 → PhaseNet(微调) 拾取 → 去重/数量控制 → P/S 到时 │
│  ↑ 全部已有可运行代码，评分器已 49 项单测覆盖                              │
└──────────────────────────────────────────────────────────────────────┘
                                    │  P/S 到时（多台站）
                                    ▼
┌─────────────────────── 决赛展示层（专家评分，需台站坐标）─────────────────┐
│  多台站震相关联 → 发震时刻反演 → 震中定位 → 震级估计 → 预警时间 → 辅助决策  │
│  ↑ 需要 stations.csv（台站经纬度）；官方数据若无位置信息，用自备/演示台网  │
└──────────────────────────────────────────────────────────────────────┘
```

## 2. 🎯 三个胜负手（先于一切工程封装）

这三件事决定名次，其余（FastAPI/Docker/压测）照做即稳。

**胜负手 A — 微调把 P 波精度压进 0.1s。** 100Hz 下 0.1s 只有 10 个采样点，预训练权重跨数据域难稳定命中。因此以 **SeisBench PhaseNet 预训练权重微调**为主线（不是从零训 TCN，也不是可选项）。训练脚手架已就位，数据一到即可开跑。

**胜负手 B — 时间对齐不能错。** 采样点下标→到时的换算一处出错，全体到时系统性偏移，模型再好也丢分。已隔离进 `utils/timing.py` 并写测试，还抓到 float64 epoch 秒精度问题（约 240ns，安全裕度对 0.1s 有约 40 万倍）。

**胜负手 C — 本地评分脚本驱动阈值优化，偏向高精确率。** "数量误差每超 1 个扣 0.5 分"是重罚，策略上宁可漏检不要多检。评分器已实现"计分容差窗内匹配 + 误报/漏检分离统计"，阈值靠它在验证集上搜索，而非拍脑袋。

## 4. 算力编排（对付免费平台的"释放"机制）

| 平台 | 角色 | 关键纪律 |
|---|---|---|
| 本地 RTX4060 8GB | 调通代码、debug（不烧云额度） | 先在本地把训练闭环跑对，再上云放量 |
| FunHPC 8G / Cloud Studio T4 16GB | 主训练、跑批实验 | 后台 nohup 跑；checkpoint 实时推 OSS |
| 阿里云 CPU ECS | 决赛期部署推理 API | 高可用；从 OSS 拉权重；自动重启+/health |
| 阿里云 OSS 桶 | 两朵云的数据/权重中转 | 训练产物"机器死前已在别处" |

三个释放陷阱及应对：Cloud Studio 关页面 10 分钟自动关机→后台跑+定期存档；80GB 硬盘 15 天清理→checkpoint 立即推 OSS；单 session 24h→存优化器状态+种子，可断点续训。微调 PhaseNet 通常 1~2 小时收敛，月额度对你是**富余**的。

## 5. 统一四周执行清单（评分驱动，按优先级）

**第 1 周 · 打通最小闭环**：拿到样例数据 → ObsPy 读通三分量 → PhaseNet 预训练直接推理 → 本地评分脚本出第一个分。**先有分，再谈优化。** 同时确认官方提交格式，填进 adapter。

**第 2 周 · 微调主模型 🎯A**：数据到位后在云 GPU 微调 PhaseNet；按事件分组切验证集（防泄漏）；用评分脚本盯"P 波命中 0.1s 比例"是否提升，而非只看 loss。

**第 3 周 · 后处理+阈值+去重 🎯C**：验证集网格搜索 P/S 阈值与合并窗口；数量控制到 5% 容差内；出稳健版/激进版两套提交参数。这周通常涨分最快。

**第 4 周 · 工程化+决赛材料**：FastAPI 服务 + Docker 部署到阿里云；准备答辩材料（架构图、误差分布图、阈值前后对比、EEW 展示层 demo、应急落地叙事）。

若时间紧，只做前三件：评分脚本复刻 → PhaseNet 微调 → 阈值/数量控制。

---

## 6. 答辩五个创新点（专家分话术）

评分感知训练（直接对齐 P±0.1s/S±0.2s 而非泛泛 loss）；预训练微调 + 评分驱动后处理的完整闭环；多台站关联的震后秒级预警扩展；每个震相输出置信度与不确定性（应急场景要知道何时不可靠）；低算力工业部署（8GB 显存可训、CPU 可推、断点续训、离线可跑）。

---

## 7. 两个仍需你确认的前提

一是**是否有带 P/S 到时的标注训练数据**（决定胜负手 A 何时能开跑）；二是**官方最终提交格式**（相对采样点还是绝对时间、CSV 字段名——只影响 adapter 一处）。这两个一定，就能把工程内核和策略叙事合并成拿来即用的完整交付。
