# PhaseNet 微调训练崩溃问题 — 求助诊断文档

## 一句话问题
用 SeisBench 的预训练 PhaseNet(stead 权重)做微调，**微调后模型完全失效**：
推理时一个 P/S 震相都拾取不到（残差 nan、评分从满分 2.0 掉到 0.0）。
需要一个能正确微调 PhaseNet 的训练脚本（或指出现有脚本的 bug）。

## 环境
- 云 GPU，Linux，CUDA 可用（单卡，约 8G 显存）
- conda 环境 `torch`，已装 `seisbench`、`torch`、`obspy`、`h5py`
- 模型：`seisbench.models.PhaseNet.from_pretrained("stead")`
- 目标任务：地震 P/S 波到时拾取，官方数据 100Hz 采样
- 评分规则：P 残差 <=0.1s 满分/1.0s 零分，S <=0.2s 满分/2.0s 零分，
  预测数量误差 >5% 每个扣 0.5 分，单条满分 2.0

## 关键诊断证据（已在 GPU 上实测）

### 证据1：预训练模型本身完全正常
微调【前】评分：平均分 2.0000/2.0，P残差 0.046s(满分率100%)，S残差 0.030s(满分率100%)。
即：未训练的模型在合成测试数据上是满分。

### 证据2：模型输出已经是 softmax 概率
对一条带 P 波的合成波形，eval 模式下 `model(x)` 的输出：
- shape = (1, 3, 3001)   # (batch, 3通道[P/S/N], 时间点)
- min = 2.4e-11, max = 1.0
- 每个时间点的 3 通道之和 = 1.0   # 说明模型内部最后一层已做 softmax
- P 通道在真实 P 波位置(1000)附近峰值 = 0.955   # 未训练时就能正确拾峰
- model.labels = "PSN"（顺序 P, S, N）

### 证据3：微调后崩溃
训练 3 epochs、40 条合成样本、batch=16、lr=1e-4、AdamW，
loss 从 1.49 -> 1.43 -> 1.44（下降后不动）。
微调【后】评分：平均分 0.0000/2.0，P/S 残差全 nan，满分率 0%。
即：训练后模型 classify() 一个峰都拾不到。

## 现有脚本的训练逻辑（怀疑有 bug 的部分）

### 软标签生成
对每条波形，按 (P到时, S到时) 生成 (3, 3001) 的目标：
- P 通道：在 P 到时处放高斯峰(sigma=20 采样点)
- S 通道：在 S 到时处放高斯峰(sigma=30)
- N(噪声)通道：= 1 - max(P, S)
- 波形做了 per-channel 零均值单位方差归一化

### 训练循环（核心，疑点所在）
```python
model = sbm.PhaseNet.from_pretrained("stead").to(device)
opt = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=1e-4)
# 类别权重：P/S 通道权重30，噪声通道权重1（为解决类别不平衡，但没用）
class_w = torch.tensor([30,30,1]).view(1,-1,1).to(device)
for ep in range(epochs):
    model.train()                      # <-- 疑点A：train模式下BatchNorm/Dropout行为
    for batch:
        xb = X[idx].to(device)         # (B,3,3001) 归一化后的波形
        yb = Y[idx].to(device)         # (B,3,3001) 软标签
        out = model(xb)                # <-- 疑点B：train模式前向是否等于eval的概率输出？
        pred = torch.softmax(out,dim=1) if (out.min()<0 or out.max()>1) else out
        loss = -(class_w * yb * torch.log(pred.clamp_min(1e-7))).sum(dim=1).mean()
        opt.zero_grad(); loss.backward(); opt.step()
```

## 我自己的怀疑方向（供参考，不一定对）
1. **BatchNorm 被带偏**：合成数据分布单一，batch 内样本高度相似，
   train() 模式下 BatchNorm 用 batch 统计量，可能几步就把内部统计冲垮。
2. **前向路径不一致**：`model(xb)` 在 train 模式经过 Dropout，
   且 SeisBench PhaseNet 的 `forward` 输出可能和 `classify()` 内部用的不是同一处理，
   导致训练目标和实际推理错位。
3. **可能根本不该手写训练循环**：SeisBench 官方有 `seisbench.generate`(sbg)
   标签生成器（如 ProbabilisticLabeller）和标准训练范式，
   与 PhaseNet 预训练方式一致。手写循环容易踩坑。
